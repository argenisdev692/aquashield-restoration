> Stack: React 19 · Inertia.js 2.0 · TypeScript 5 · TanStack Query v5 · TanStack Table v8 · Tailwind CSS · shadcn/ui

---

## Layer Responsibilities

### `common/` — Generic primitives
Framework-agnostic UI components, helpers, and hooks. No domain knowledge.
- Rule: Cannot import from `modules/` or `pages/`

### `modules/` — Domain-specific shared code
Business-domain components and TanStack Query hooks used across multiple pages.
- Rule: Cannot import from `pages/`. May import from `common/` and other modules via `types.ts`

### `pages/` — Inertia Page components
One directory per route group. Only place that uses `usePage()` and `useForm()` from Inertia.
- Rule: Imports from `modules/` and `common/`. Never the reverse

### `shadcn/` — UI library primitives
Auto-generated by shadcn CLI. Never hand-edited.

### `types/` — TypeScript contracts
Single source of truth for interfaces. Must mirror backend DTO field names exactly.

## Inertia Page Component Pattern

**Rules:**
- Always `export default` — Inertia requires it
- Always include `<Head title="..." />` for SEO
- Layout wraps page content
- Explicit return type `React.JSX.Element`
- Typed via `usePage<PagePropsInterface>()`

---

## TanStack Query Hook Pattern

**Rules:**
- Always type generics: `useQuery<TData, TError>`
- Query key hierarchy: `['context', 'operation', ...params]`
- `placeholderData: keepPreviousData` for paginated lists
- Errors typed as `Error` — never `unknown`

---

## TanStack Mutation Hook Pattern

**Rules:**
- Always invalidate queries after mutation in `onSuccess`
- Type generics: `useMutation<TData, TError, TVariables>`

---

## TanStack Table Pattern

**Rules:**
- Generic `DataTable` lives in `common/data-table/`
- Memoize column definitions with `useMemo`
- Use `flexRender()` for all cell rendering
- Soft deletes: rows with `deleted_at` show View + Restore only
- Active rows show View + Edit + Delete
- Use `DeleteConfirmModal` instead of `window.confirm()`

---

## TypeScript Types Structure

**Rules:**
- `types/inertia.d.ts` — Inertia page props augmentation
- `types/api.ts` — API response contracts (mirrors backend DTOs)
- `types/props.ts` — Shared React prop utility types
- All types must match backend DTO field names exactly

---

## Import Conventions

Use absolute path aliases — never relative `../../` beyond 1 level:

```ts
// ✅ Correct
import { useUsers } from "@/modules/users/hooks/useUsers";
import type { UserListItem } from "@/types/api";

// ❌ Wrong
import { useUsers } from "../../../modules/users/hooks/useUsers";
```

**Import order:**
1. Node built-ins
2. External libraries
3. Internal `@/` imports
4. Relative `./ ../` (same directory only)

---

## Component Rules

- One component per file
- Named exports except Inertia Pages (require `export default`)
- Props via `interface` — always `interface FooProps`
- Explicit return types — `React.JSX.Element` or `React.ReactNode`
- `function` declaration over `const` arrow for components
- Shadcn UI first — check existing components before creating new ones
- `className` and `children` always last in props

---

## State Management Decision Tree

```
Server data? → TanStack Query
Page-level initial state from Laravel? → Inertia usePage().props
User's in-progress form? → Inertia useForm() or useMutation
UI state (open/closed, filters)? → useState / useReducer
UI state needs to survive navigation? → Inertia useRemember()
State shared across unrelated components? → React Context (sparingly)
```

**Golden Rule:** Never duplicate data in both Inertia props and TanStack Query cache.

## CSS & Styling

- Tailwind only — no inline styles
- All design tokens in `app.css` as CSS custom properties
- Follow `rules-styles.md` for token structure
- Use `cn()` helper for conditional class merging
- Never use arbitrary Tailwind values like `bg-[#1a1a1a]`

---

## UI / UX Patterns

- Icons: Always use `lucide-react`
- Interactive elements: Must have `cursor: pointer`
- Data tables: Text and actions centered
- Active navigation: Distinct background, border, rounded corners

---

## Routing Architecture

For complete route structure and patterns, see `RUTAS-REFERENCIA.md`.

**Key Rules:**
- Web routes: `/users/*` (Inertia pages) + `/users/data/admin/*` (JSON for React Query)
- API routes: `/api/users/admin/*` (mobile/external)
- Never call `/api/*` from Inertia pages
- Web: session auth, API: Sanctum tokens

## DataTableDateRangeFilter — Usage

**Rules:**
- Both `dateFrom` and `dateTo` are optional
- Client-side validation: `dateFrom` ≤ `dateTo`
- Dates as ISO 8601 strings `YYYY-MM-DD`
- Wrap onChange in `useTransition()`
- Must be included in every index page
- Must be part of TanStack Query key

---

## ExportButton — Usage

**Rules:**
- Must be included in every index page
- Passes current active filters (including date range) to export endpoints
- Supports Excel and PDF formats
